# S2-1 Capacitor Android Base

## 목표
- 현재 웹 앱에 Capacitor 기반 Android 셸을 추가한다.
- Android 앱은 원격 URL이 아니라 빌드된 로컬 웹 자산(`dist`)을 포함해 실행한다.
- 이후 S2-2 Android 인증 구현이 가능한 최소 Android 프로젝트 구조와 npm 스크립트를 제공한다.

## 전제
- 현재 저장소는 Vite + React 웹 앱이며 Android 네이티브 구조가 없다.
- Android package/application id는 `com.jwjh.wedding`을 사용한다.
- 앱 이름은 `Wedding Tour`를 사용한다.
- `APP_URL`은 웹 배포용 값으로 유지하되 Android runtime asset source로는 사용하지 않는다.

## 산출물
- `capacitor.config.ts`
- `android/` 프로젝트 구조
- `package.json` Android 관련 스크립트
- `.gitignore` Android/Capacitor 로컬 부산물 반영
- `tests/unit/capacitor-config.test.ts`
- `/plan` 진행 로그

## 구현 규칙
- 의존성 추가:
  - `@capacitor/core@7.6.1`
  - `@capacitor/cli@7.6.1`
  - `@capacitor/android@7.6.1`
- 현재 로컬 Node 버전이 20.9.x 이므로, Node 22 이상이 필요한 Capacitor 8 계열은 사용하지 않는다.
- `capacitor.config.ts`는 다음 값을 가진다.
  - `appId: "com.jwjh.wedding"`
  - `appName: "Wedding Tour"`
  - `webDir: "dist"`
  - `bundledWebRuntime: false`
- `package.json`에는 다음 스크립트를 추가한다.
  - `android:sync`
  - `android:open`
  - `android:run`
  - `android:build:web`
  - `android:prepare`
- `android:prepare`는 `npm run build && npx cap sync android`를 수행한다.
- Android 프로젝트는 Capacitor 기본 생성 결과를 포함하되, 인증/서명/릴리스 설정은 S2-2/S2-3에서 다룬다.

## 검증 기준
- `tests/unit/capacitor-config.test.ts`가 통과한다.
- `npm run lint`가 통과한다.
- `npm run build`가 통과한다.
- `npm run android:prepare`가 통과한다.
- `android/` 디렉터리가 생성되어 있어야 한다.
