# S3-8 Map Viewport Fit

## 목표
- 지도 화면에서 지도, 선택 탭, 상세 정보 카드가 Android 화면 높이 안에서 자연스럽게 배치되도록 한다.
- 지도 화면의 상세 카드가 하단 탭바 아래로 밀리지 않도록 한다.

## 원인
- 현재 지도 높이는 고정 높이 기반이라 어떤 화면에서는 여유 공간이 남고, 어떤 화면에서는 상세 카드가 fold 아래로 밀린다.
- 지도 페이지 루트가 남은 뷰포트 높이를 기준으로 flex 배치를 하지 않아 실제 화면 크기에 유연하게 맞지 않는다.

## 구현 규칙
- 지도 화면은 `min-h-full` 기반의 세로 flex 레이아웃을 사용한다.
- sticky header와 선택 탭 아래의 본문 영역이 남은 높이를 채우도록 구성한다.
- 지도 카드 영역은 `flex-1`로 남은 공간을 채우고, 최소 높이만 가진다.
- 상세 카드와 지도는 같은 화면 안에서 함께 보여야 하며 하단 탭바 아래로 밀리지 않아야 한다.
- 고정 높이 `h-[min(...)]` 방식은 제거한다.

## 검증 기준
- `Map.tsx`가 `min-h-full` / `flex-1` / `min-h-0` 구조를 가진다.
- `Map.tsx`는 더 이상 `h-[min(58vh,34rem)]` 고정 높이를 사용하지 않는다.
- `Map.tsx`의 선택 웨딩홀 상세 카드가 지도 아래 별도 영역으로 유지된다.
- `npm test -- tests/unit/map-view-scale.test.ts`가 통과한다.
