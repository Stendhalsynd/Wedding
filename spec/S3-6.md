# S3-6 Map View Scale And Focus

## 목표
- 지도 뷰에서 선택된 웨딩홀의 지도가 모바일에서도 충분히 크게 보이도록 한다.
- 지도 위에 겹쳐지는 정보 카드 때문에 실제 지도 영역이 가려지는 문제를 줄인다.

## 원인
- 현재 지도 카드 안에 상세 정보 버튼이 `absolute`로 겹쳐져 있어 지도 면적이 크게 줄어든다.
- 지도 컨테이너가 `flex-1`에만 의존해 실제 모바일 화면에서 충분한 높이를 확보하지 못한다.

## 구현 규칙
- 지도 iframe 컨테이너는 모바일에서도 충분한 높이를 가지는 명시적 높이 규칙을 가져야 한다.
- 선택된 웨딩홀의 정보 카드는 지도 위 오버레이가 아니라 지도 아래 별도 카드로 배치한다.
- 지도 뷰는 한 웨딩홀에 집중해서 위치를 확인할 수 있는 구조여야 한다.
- 지도 선택 탭은 유지하되 지도 본문보다 시각적 우선순위가 낮아야 한다.

## 검증 기준
- `Map.tsx`에 명시적인 대형 지도 높이 클래스가 존재한다.
- `Map.tsx`는 선택 웨딩홀 정보 버튼을 지도 아래 별도 카드로 렌더링한다.
- `Map.tsx`는 더 이상 지도 카드 내부에 `absolute bottom-4 left-4 right-4` 오버레이를 두지 않는다.
- `npm test -- tests/unit/map-view-scale.test.ts`가 통과한다.

## 제외 범위
- 지도 provider 교체
- 지도 좌표 기반 정밀 줌 제어
- 여러 웨딩홀을 동시 표시하는 집계형 지도 리디자인
