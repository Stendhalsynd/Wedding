# S2-5 Android Release Publish

## 목표
- signed release APK를 실제로 생성한다.
- 최신 커밋을 원격에 푸시하고, 버전 태그와 GitHub Release를 생성한다.
- release APK를 GitHub Release asset으로 업로드하고 Discord 공지까지 연결한다.

## 전제
- S2-3 release signing 구성이 구현되어 있다.
- 실제 keystore와 `google-services.json`이 로컬에 준비되어 있다.
- GitHub token과 Discord webhook이 로컬 환경에 준비되어 있다.

## 구현 규칙
- APK 빌드 전 `npm test`, `npm run lint`, `npm run build`, `npm run android:prepare`를 통과해야 한다.
- release 태그는 `v<package.json.version>` 형식을 사용한다.
- GitHub Release에는 release APK만 업로드한다.
- `google-services.json`과 `.env`는 로컬 빌드 전용으로 유지하고 저장소에 커밋하지 않는다.

## 검증 기준
- `dist/releases/Wedding-v<version>-release.apk`가 생성된다.
- `adb install` 없이도 파일 존재와 Gradle release task 성공으로 signed build를 입증할 수 있어야 한다.
- `git push origin main`이 통과한다.
- GitHub Release 생성 또는 업데이트가 통과한다.
- Discord webhook 전송이 성공한다.

## 제외 범위
- Play Store 업로드
- AAB 생성
