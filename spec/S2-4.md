# S2-4 Web / Android App Identity Icons

## 목표
- 웹 탭의 대표 아이콘(`favicon.ico`)을 추가한다.
- Android 앱 설치 후 보이는 launcher icon을 브랜드에 맞는 대표 이미지로 교체한다.
- 웹과 Android는 같은 마스터 아이콘 컨셉을 공유한다.

## 전제
- 현재 저장소에는 웹용 favicon이 없다.
- Android 기본 Capacitor launcher icon이 생성되어 있다.
- 아이콘은 이 프로젝트의 웨딩 서비스 정체성을 드러내야 하며, 작은 크기에서도 인식 가능해야 한다.
- 대표 아이콘은 `public/예비신랑.png`, `public/예비신부.png`를 공용 소스로 사용한다.

## 산출물
- `scripts/generate-app-icons.py`
- `public/favicon.ico`
- `public/favicon-32x32.png`
- `public/apple-touch-icon.png`
- `public/icon-512.png`
- Android `mipmap-*` launcher icon 교체
- `android/app/src/main/res/values/ic_launcher_background.xml` 색상 조정
- `tests/unit/app-icons.test.ts`
- `index.html` favicon / apple-touch-icon 링크

## 구현 규칙
- 하나의 마스터 아이콘 컨셉에서 웹/Android 자산을 파생한다.
- favicon은 `ico` 파일을 포함해야 한다.
- Android adaptive icon foreground/background 구조는 유지한다.
- launcher icon은 작은 크기에서도 읽히도록 텍스트 없이 심볼 중심으로 만든다.
- 웹/Android 아이콘 생성은 스크립트로 재현 가능해야 한다.
- 원본 역할 이미지의 얼굴/상반신 일러스트만 사용하고, 하단 텍스트는 대표 아이콘에 포함하지 않는다.

## 검증 기준
- `tests/unit/app-icons.test.ts`가 통과한다.
- `npm run build`가 통과한다.
- `npm run android:prepare`가 통과한다.
- `public/favicon.ico`가 존재한다.
- Android `mipmap-*`의 `ic_launcher.png`, `ic_launcher_round.png`, `ic_launcher_foreground.png`가 모두 존재한다.
