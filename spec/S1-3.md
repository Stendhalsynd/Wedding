# S1-3 APK 릴리스 업로드 및 Discord 배포 규칙

## 목표
- 이 프로젝트에서도 `RoutineMate`처럼 릴리스 태그 기준으로 릴리스 노트를 만들고, release APK를 GitHub Release에 업로드할 수 있어야 한다.
- 릴리스 완료 후 Discord webhook으로 변경 요약을 전송할 수 있어야 한다.

## 전제
- 현재 `Wedding` 리포지토리에는 Android/APK 빌드 구조가 없다.
- 따라서 이번 슬라이스는 "이미 빌드된 release APK 경로를 입력받아 업로드/공지"하는 자동화까지를 범위로 한다.

## 입력 규칙
- GitHub Release 업로드 스크립트 입력:
  - `tag`
  - `apk_path`
  - 선택 `title`
- Discord 전송 스크립트 입력:
  - `title`
  - `version`
  - `commit`
  - `release_notes_file` 또는 요약 텍스트

## 동작 규칙
- release 업로드 스크립트는 다음을 보장한다.
  - APK 파일 존재 확인
  - `preview` / `debug` APK 업로드 거부
  - `release` 문자열이 포함된 APK만 허용
  - 릴리즈 노트 파일 생성
  - 태그가 이미 있으면 asset upload, 없으면 release create
- Discord 전송 스크립트는 다음을 보장한다.
  - `~/StudioProjects/dash/.env`에서 webhook 로드
  - `DISCORD_WEBHOOK_URL` → `OMC_DISCORD_WEBHOOK_URL` 순서 fallback
  - HTTP 204 성공 검증
  - 프로젝트명, 커밋, 변경 요약, 버전, 릴리스 URL 포함

## 산출물
- `scripts/generate-release-notes.mjs`
- `scripts/release-apk.sh`
- `scripts/release-discord.sh`
- `package.json` release 관련 명령
- release guard 테스트

## 검증 기준
- release guard 테스트가 preview/debug APK를 거부한다.
- release guard 테스트가 release APK를 dry-run에서 통과시킨다.
- 릴리즈 노트 파일이 생성된다.
- Discord 스크립트는 webhook 누락 시 명확한 에러를 낸다.
