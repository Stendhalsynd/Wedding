# S3-4 Firestore User Rules Sync

## 목표
- 현재 앱이 쓰는 사용자 문서 필드와 Firestore rules를 일치시킨다.
- 첫 로그인, 역할 건너뛰기, 역할 설정 업데이트가 rules에 의해 막히지 않도록 한다.

## 원인
- 앱은 `roleSetupDismissedAt` 필드를 쓰지만 Firestore rules의 `isValidUser` 허용 필드 목록에는 포함되어 있지 않다.
- 이 상태에서는 사용자 문서 create/update가 `permission-denied`로 실패할 수 있다.

## 구현 규칙
- `users` 문서 허용 필드 목록에 `roleSetupDismissedAt`를 추가한다.
- `roleSetupDismissedAt`는 `null`, `timestamp`, 또는 필드 부재를 허용한다.
- 기존 `role`, `profileImageUrl`, `coupleId` 규칙은 유지한다.

## 검증 기준
- rules 파일이 `roleSetupDismissedAt`를 허용한다.
- 관련 테스트가 통과한다.

## 제외 범위
- weddingHalls, couples rules 리팩터링
- Rules 배포 자동화 전체 구축
