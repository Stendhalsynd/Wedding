# S1-1 사용자 역할 선택 및 프로필 이미지 규칙

## 목표
- 사용자는 첫 로그인 이후 예비신부(`bride`) 또는 예비신랑(`groom`) 역할을 선택할 수 있어야 한다.
- 사용자는 역할 선택을 건너뛸 수 있어야 하며, 이후에도 앱 사용은 가능해야 한다.
- 사용자가 역할을 선택하거나 설정에서 변경하면 프로필 이미지는 로컬 정적 이미지로 일관되게 표시되어야 한다.

## 상태 모델
- `users.role`
  - 값: `bride | groom | null`
  - 역할이 정해지지 않은 경우 `null`
- `users.profileImageUrl`
  - 값: `string | undefined`
  - 역할이 있으면 `DEFAULT_PROFILE_IMAGES[role]` 경로를 사용한다.
- `users.roleSetupDismissedAt`
  - 값: Firestore timestamp 또는 null
  - 사용자가 최초 역할 설정 화면에서 건너뛰기를 선택한 경우 기록한다.

## 화면 동작
- 로그인 후 사용자 문서가 없으면 기본 사용자 문서를 생성한다.
- 로그인 후 `role === null` 이고 `roleSetupDismissedAt`가 없으면 `/profile-setup`으로 이동한다.
- 로그인 후 `role === null` 이고 `roleSetupDismissedAt`가 있으면 메인 진입을 허용한다.
- `ProfileSetup`
  - 예신/예랑 버튼은 각 정적 이미지 미리보기를 보여준다.
  - `시작하기`를 누르면 `role`, `profileImageUrl`를 기록하고 `roleSetupDismissedAt`는 제거 또는 null 처리한다.
  - `나중에 설정하기`를 누르면 `role`은 유지하지 않고 `roleSetupDismissedAt`를 기록한 뒤 메인으로 이동한다.
- `Settings`
  - 역할 변경 시 `role`, `profileImageUrl`를 함께 갱신한다.
  - 역할이 있으면 정적 이미지를 우선 표시한다.
  - 역할이 없고 이미지가 없으면 현재 fallback UI를 유지한다.

## Fallback 규칙
- 기존 구형 원격 아바타 URL을 가진 사용자는 앱 진입 시 정적 이미지로 마이그레이션한다.
- 역할이 없는 사용자는 기본 프로필 이미지를 강제하지 않는다.

## 검증 기준
- 첫 로그인 사용자는 역할 설정 화면을 본다.
- 역할 선택 후 메인 진입이 가능하다.
- 건너뛰기 후에도 메인 진입이 가능하다.
- 설정에서 역할 변경 시 프로필 이미지가 즉시 바뀐다.
