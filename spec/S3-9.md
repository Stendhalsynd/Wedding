# S3-9 Vercel SPA Route Rewrites

## 목표
- Vercel production에서 `/login`, `/map`, `/compare`, `/settings`, `/hall/:id` 같은 직접 진입 URL이 404 없이 동작하도록 한다.

## 원인
- 현재 배포에는 SPA rewrite 설정이 없어 Vercel이 React Router 경로를 정적 파일 경로로 처리한다.

## 구현 규칙
- `vercel.json`에 모든 비정적 경로를 `/index.html`로 보내는 rewrite를 추가한다.
- 이미지, 정적 asset, vite build output을 깨지 않도록 일반적인 SPA fallback 형식을 사용한다.

## 검증 기준
- `vercel.json`이 존재한다.
- `vercel.json`에 `source: "/(.*)"` 또는 동등한 SPA fallback rewrite가 존재한다.
- `npm test -- tests/unit/vercel-spa-routing.test.ts`가 통과한다.

## 제외 범위
- 서버사이드 렌더링 도입
- edge function / server function 추가
