# S3-12 HallDetail Budget Threshold Indicators

## 목표
- 웨딩홀 상세 화면에서 대관료와 식대 입력 범위를 현실적으로 넓힌다.
- 사용자가 예산 기준을 넘긴 값을 입력해도 저장할 수 있어야 한다.
- 예산 기준 초과 여부를 슬라이더 색상과 상태 문구로 즉시 식별할 수 있게 한다.

## 구현 규칙
- 대관료 슬라이더 최대값은 `1000`으로 한다.
- 식대 슬라이더 최대값은 `10`으로 한다.
- 대관료 예산 기준은 `750`이다.
- 식대 예산 기준은 `7.5`이다.
- 예산 기준 이하일 때는 기본 rose 계열 표시를 유지한다.
- 예산 기준 초과 시 슬라이더 accent와 상태 badge를 amber 계열로 바꾸고 `예산 초과` 문구를 표시한다.
- 이 표시는 `HallDetail.tsx`의 견적 정보 영역에만 적용한다.

## 검증 기준
- `HallDetail.tsx`가 `RENTAL_FEE_MAX = 1000`, `MEAL_FEE_MAX = 10` 기준을 사용한다.
- `HallDetail.tsx`가 `RENTAL_FEE_BUDGET_LIMIT = 750`, `MEAL_FEE_BUDGET_LIMIT = 7.5` 기준을 사용한다.
- `HallDetail.tsx`에 `예산 초과` 표시와 amber accent 전환 로직이 존재한다.
- `npm test -- tests/unit/hall-detail-budget-thresholds.test.ts`가 통과한다.
