# S1-2 웨딩홀 작성자 표시 규칙

## 목표
- 연인 연결 후 각 웨딩홀 카드와 상세 화면에서 누가 추가한 기록인지 바로 식별할 수 있어야 한다.
- 작성자에게 역할이 있으면 `이미지 + 예신/예랑`으로 표시한다.
- 작성자 역할이 없으면 현재처럼 이름 첫 글자 기반 아바타를 유지한다.

## 데이터 기준
- `weddingHalls.authorId`를 작성자 식별의 기준으로 사용한다.
- `weddingHalls.authorName`은 fallback 텍스트에만 사용한다.
- 작성자 표시용 역할/프로필 정보는 `users/{authorId}` 문서에서 읽는다.

## 표시 규칙
- 작성자 역할이 `bride`
  - 원형 프로필: `/예비신부.png`
  - 라벨: `예신`
- 작성자 역할이 `groom`
  - 원형 프로필: `/예비신랑.png`
  - 라벨: `예랑`
- 작성자 역할이 없거나 사용자 문서를 조회하지 못한 경우
  - 현재와 같은 이니셜 원형 아바타 표시
  - 이름 fallback: `authorName`, 없으면 `나`

## 적용 범위
- `Dashboard`
  - 웨딩홀 리스트 카드의 작성자 표시 영역
- `HallDetail`
  - 상단 메타 정보 또는 헤더 영역의 작성자 표시

## 비범위
- `Compare`, `Map` 화면 작성자 표시 확장은 이번 작업 범위에서 제외한다.
- 작성자 역할을 웨딩홀 문서에 별도로 중복 저장하지 않는다.

## 검증 기준
- 역할이 있는 작성자는 카드와 상세에서 `이미지 + 예신/예랑`으로 표시된다.
- 역할이 없는 작성자는 기존 이니셜 표시를 유지한다.
- 역할을 설정에서 변경하면 기존 웨딩홀도 새 역할 기준으로 표시된다.
