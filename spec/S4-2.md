# S4-2 HappyLife Secure Planning Integration

## 목표
- HappyLife의 예산, 업체, 주거, 정책, 타임라인 데이터를 Wedding 앱의 Google 로그인 및 커플 공유 모델 안으로 통합한다.
- 로그인 전에는 민감한 예산/후보 데이터가 보이지 않아야 한다.
- 커플 연결 전에는 `user.uid`, 연결 후에는 `coupleId` 기준으로 같은 데이터를 공유한다.

## 구현 규칙
- Firestore 컬렉션은 `planningBudgetRows`, `planningVendors`, `planningApartments`, `planningPolicies`, `planningTimeline`, `planningMetadata`를 사용한다.
- 각 planning 문서는 `id`, `coupleId`, `authorId`, `createdAt`, `updatedAt`을 가진다.
- 첫 planning 진입 시 현재 owner key(`coupleId || user.uid`)에 seed 데이터를 1회 생성한다.
- 커플 연결 시 두 사용자의 solo planning 문서를 새 `coupleId`로 batch migration한다.
- 기존 Wedding 앱의 Firebase Auth, Firestore, `isPartOfCouple` rules 패턴을 재사용한다.
- 1차 버전은 커플 외부 공유, 게스트 링크, 공개 페이지를 만들지 않는다.

## 검증 기준
- 비로그인 사용자는 planning 화면에 접근할 수 없다.
- `MobileLayout`에 관리 탭이 있고 `/planning` 라우트가 보호된다.
- Firestore rules는 planning 컬렉션을 인증 사용자 본인 또는 커플에게만 허용한다.
- seed 초기화는 중복 생성되지 않도록 metadata 문서를 사용한다.
- `npm test -- tests/unit/planning-data-model.test.ts tests/unit/planning-firestore-rules.test.ts tests/unit/planning-ui-integration.test.ts`가 통과한다.
