# S3-13 Couple Connection Batch Rules

## 목표
- 기존 솔로 사용자가 가진 웨딩홀 문서를 새 커플 `coupleId`로 옮길 때 Firestore rules가 배치 쓰기를 허용해야 한다.
- 연결 실패 시 권한 오류를 사용자에게 더 구체적으로 보여준다.

## 구현 규칙
- `weddingHalls` update rules는 기존 `coupleId == request.auth.uid` 인 문서를 새 `coupleId`로 이동하는 배치 쓰기를 허용해야 한다.
- 이 허용은 현재 요청 안에서 생성되거나 존재하는 `couples/{newCoupleId}` 문서를 기준으로 검증해야 한다.
- `ConnectCouple.tsx`는 `permission-denied`를 일반 오류와 구분해 표시한다.

## 검증 기준
- `firestore.rules`가 `existsAfter` 또는 `getAfter` 기반으로 연결 중인 커플 문서를 검증한다.
- `ConnectCouple.tsx`가 `permission-denied` 분기 메시지를 가진다.
- `npm test -- tests/unit/firestore-rules-couple-connection.test.ts tests/unit/compare-sorting-behavior.test.ts` 이전에 rules 관련 정적 검증이 통과한다.
