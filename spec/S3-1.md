# S3-1 Secret Hygiene and Push Protection

## 목표
- Firebase 웹 앱 설정이 저장소에 평문 JSON으로 커밋되지 않도록 한다.
- Google API 키나 유사한 공개/비공개 키가 푸시 직전에 차단되도록 한다.
- 개발자가 동일한 실수를 반복하지 않도록 로컬 훅과 문서 기준을 같이 제공한다.

## 범위
- `firebase-applet-config.json` 제거
- Firebase 설정을 `VITE_*` 환경 변수 기반으로 전환
- `.env.example` 업데이트
- 비밀 값 검사용 스크립트 추가
- `pre-push` 훅 추가 및 로컬 저장소에 연결
- README에 운영 규칙 문서화

## 구현 규칙
- 실제 키 값은 git tracked 파일에 저장하지 않는다.
- Firebase 웹 설정은 `import.meta.env`만 사용한다.
- 예제 파일에는 placeholder만 넣는다.
- 푸시 차단은 파일명 기반 차단과 패턴 기반 차단을 함께 사용한다.
- 검사 스크립트는 현재 변경분만이 아니라 tracked text file 전체를 검사할 수 있어야 한다.

## 차단 대상
- `firebase-applet-config.json`
- Google API key 패턴 `AIza...`
- `.env` 계열 파일의 tracked 상태

## 검증 기준
- `src/firebase.ts`는 더 이상 `firebase-applet-config.json`을 import 하지 않는다.
- `firebase-applet-config.json`은 저장소 기준 파일로 존재하지 않는다.
- `npm run secrets:check`가 현재 워크트리에서 통과한다.
- `.githooks/pre-push`가 `npm run secrets:check`를 호출한다.
- `git config core.hooksPath .githooks`가 이 저장소에 적용된다.

## 제외 범위
- 이미 공개된 git history rewrite
- GitHub organization 수준 secret scanning 정책
- 서버 측 secret manager 도입
