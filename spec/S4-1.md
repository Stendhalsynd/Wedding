# S4-1 Versioned Release Publish

## 목표
- 동일 태그 재업로드가 아니라 새 버전으로 APK 릴리스를 발행한다.
- `package.json` 버전, Android `versionName`/`versionCode`, Git tag, GitHub Release, Discord 공지가 일관되게 연결되게 한다.

## 구현 규칙
- `package.json.version`을 새 버전으로 올린다.
- 새 Git tag는 `v<package.json.version>` 형식을 사용한다.
- 새 APK artifact 이름도 새 버전을 반영해야 한다.
- GitHub Release는 새 태그 기준으로 생성한다.
- Discord 공지도 새 태그 기준으로 발송한다.

## 검증 기준
- `package.json.version`이 이전 `0.0.0`보다 증가한다.
- release build 결과물이 새 버전 파일명으로 생성된다.
- 새 tag와 새 GitHub Release가 존재한다.

## 제외 범위
- Play Store 업로드
- AAB 배포
