# S3-2 Mobile Bottom Navigation Safe-Area Fix

## 목표
- 모바일 Chrome/Safari에서 하단 네비게이션이 브라우저 UI 아래로 밀리지 않도록 한다.
- 하단 네비게이션이 항상 화면에 보이고 터치 가능해야 한다.
- 콘텐츠 스크롤 영역이 하단 네비게이션과 겹치지 않도록 한다.

## 원인
- 현재 `MobileLayout`은 `h-screen` 컨테이너 내부에 `absolute bottom-0` 네비게이션을 두고 있다.
- 모바일 브라우저의 동적 주소창과 safe area는 고정 `100vh`/`h-screen`과 어긋날 수 있다.
- 그 결과 실제 보이는 뷰포트보다 더 아래에 네비게이션이 배치되어 터치 불가 상태가 생긴다.

## 구현 규칙
- 레이아웃 높이는 `100vh` fallback + `100svh` + `100dvh`를 지원해야 한다.
- 하단 네비게이션은 컨테이너 절대배치가 아니라 뷰포트 기준 `fixed bottom-0`으로 배치한다.
- 하단 패딩은 `env(safe-area-inset-bottom)`를 포함해야 한다.
- 스크롤 콘텐츠는 네비게이션 높이 + safe area 만큼 하단 여백을 확보해야 한다.

## 검증 기준
- `MobileLayout.tsx`는 더 이상 `absolute bottom-0` 네비게이션을 사용하지 않는다.
- `MobileLayout.tsx`는 safe-area 하단 패딩과 뷰포트 고정 네비게이션 구조를 가진다.
- `index.css`에는 앱 뷰포트 높이/하단 safe-area 유틸리티가 존재한다.
- `npm test -- tests/unit/mobile-layout-safe-area.test.ts`가 통과한다.
- `npm run lint`와 `npm run build`가 통과한다.

- 태블릿 전용 네비게이션 레이아웃 분기
