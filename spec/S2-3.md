# S2-3 Android Signing And Release Build

## 목표
- 로컬 환경에서 signed release APK를 생성할 수 있어야 한다.
- Android versionName/versionCode는 `package.json.version`에서 파생되어야 한다.
- keystore 비밀값은 저장소에 포함하지 않고 `.env`와 로컬 생성 파일로만 관리한다.

## 전제
- S2-1 Capacitor Android base가 존재한다.
- S2-2 auth bridge가 구현되어 있다.
- 현재 `package.json.version`은 Android 버전의 single source of truth다.
- `0.0.0`처럼 계산 결과가 0인 버전도 허용하되, Android `versionCode`는 최소 1이어야 한다.

## 산출물
- `scripts/android-version.mjs`
- `scripts/release-android-build.sh`
- `tests/unit/android-release-build.test.ts`
- `android/app/build.gradle` release signing 및 version metadata 반영
- `.env.example` Android signing 변수 추가
- `.gitignore`에 `android/keystore.properties` 반영
- `package.json` Android release build 스크립트 추가
- `/plan` 진행 로그

## 구현 규칙
- 환경 변수:
  - `ANDROID_KEYSTORE_PATH`
  - `ANDROID_KEYSTORE_PASSWORD`
  - `ANDROID_KEY_ALIAS`
  - `ANDROID_KEY_PASSWORD`
  - 선택 `ANDROID_KEYSTORE_TYPE`
- `scripts/android-version.mjs`
  - `package.json.version`을 읽는다.
  - `major*10000 + minor*100 + patch`로 versionCode를 계산한다.
  - 계산 결과가 1 미만이면 1을 사용한다.
- `android/app/build.gradle`
  - `package.json.version`을 읽어 `versionName`에 반영한다.
  - 계산된 `versionCode`를 반영한다.
  - `android/keystore.properties`가 있을 때 release signing config를 사용한다.
- `scripts/release-android-build.sh`
  - signing 환경 변수 존재 여부 검증
  - `android/keystore.properties` 생성
  - `npm test`
  - `npm run lint`
  - `npm run android:prepare`
  - `cd android && ./gradlew assembleRelease`
  - `android/app/build/outputs/apk/release/app-release.apk`를 `dist/releases/Wedding-v<version>-release.apk`로 복사
- `ANDROID_BUILD_DRY_RUN=1`이면 위 과정을 실제 빌드 없이 검증만 수행한다.

## 검증 기준
- `tests/unit/android-release-build.test.ts`가 통과한다.
- `npm run lint`가 통과한다.
- `scripts/release-android-build.sh` dry-run이 통과한다.
- 임시 keystore로 `./gradlew assembleRelease`가 통과하고 release APK가 생성된다.
