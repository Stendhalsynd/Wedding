# S2-2 Android Google Login Bridge

## 목표
- Android 앱에서는 `signInWithPopup` 대신 네이티브 Google 로그인을 사용한다.
- 네이티브 로그인 결과의 Google credential을 Firebase JavaScript SDK 세션으로 연결한다.
- 웹 브라우저에서는 기존 `signInWithPopup` 흐름을 유지한다.

## 전제
- S2-1로 Capacitor Android 기반이 추가되어 있다.
- Firebase JavaScript SDK의 `auth`, `onAuthStateChanged`, Firestore 흐름은 계속 canonical auth state로 유지한다.
- Android package id는 `com.jwjh.wedding`이다.

## 구현 규칙
- 의존성 추가:
  - `@capacitor-firebase/authentication@7.5.0`
- Android 분기는 `Capacitor.getPlatform() === "android"` 그리고 `Capacitor.isNativePlatform()`을 모두 만족할 때만 활성화한다.
- Android에서는 `FirebaseAuthentication.signInWithGoogle({ skipNativeAuth: true })`를 사용한다.
- 네이티브 로그인 성공 후:
  1. `idToken`, `accessToken` 추출
  2. `GoogleAuthProvider.credential(idToken, accessToken ?? null)` 생성
  3. `signInWithCredential(auth, credential)` 호출
- `idToken`이 없으면 명시적으로 오류를 발생시킨다.
- 로그아웃은 Android에서 네이티브 플러그인 로그아웃과 Firebase JS SDK 로그아웃을 모두 수행한다.
- Capacitor config에는 다음 plugin 설정을 추가한다.
  - `plugins.FirebaseAuthentication.skipNativeAuth = true`
  - `plugins.FirebaseAuthentication.providers = ["google.com"]`

## UI 규칙
- Android 네이티브 런타임에서는 인앱 브라우저 경고 UI를 표시하지 않는다.
- 웹에서만 `unauthorized-domain`, `popup-closed-by-user`, 외부 브라우저 안내를 유지한다.
- Android 오류는 일반적인 로그인 실패 메시지와 credential 누락 메시지로 처리한다.
