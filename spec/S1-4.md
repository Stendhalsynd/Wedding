# S1-4 Vercel 의존성 설치 정책

## 목표
- Vercel 배포 시 `npm install` 단계가 React 19와 `pannellum-react`의 peer dependency 충돌로 실패하지 않아야 한다.
- 로컬 개발 환경과 Vercel 배포 환경의 의존성 해석 규칙이 동일해야 한다.

## 문제 정의
- `pannellum-react@1.2.4`는 `react@16.x` peer dependency를 선언한다.
- 현재 프로젝트는 `react@19.x`를 사용한다.
- 로컬에서는 `npm install --legacy-peer-deps`로 설치했기 때문에 통과했지만, Vercel은 기본 `npm install`을 사용해 실패한다.

## 결정
- 저장소 루트에 `.npmrc`를 추가해 `legacy-peer-deps=true`를 명시한다.
- Vercel은 기본 `npm install`에서 이 설정을 읽어 동일한 정책으로 설치한다.
- 이번 슬라이스에서는 `pannellum-react` 자체를 교체하지 않는다. 해당 교체는 별도 스펙 슬라이스로 분리한다.

## 산출물
- 루트 `.npmrc`
- `/plan` 진행 로그 업데이트

## 검증 기준
- 깨끗한 환경에서 `npm install`이 추가 플래그 없이 통과한다.
- `npm run build`가 통과한다.
- Vercel CLI 또는 Vercel 배포 환경에서 install 단계가 더 이상 `ERESOLVE`로 실패하지 않는다.

## 리스크
- `legacy-peer-deps`는 충돌을 무시하는 정책이므로 장기적으로는 `pannellum-react` 대체 검토가 필요하다.
- 이번 슬라이스의 목적은 배포 복구이며, 라이브러리 교체는 포함하지 않는다.
