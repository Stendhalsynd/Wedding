# S3-5 Tour Header And Bottom Action Layout

## 목표
- 투어 화면의 제목 영역이 스크롤 중에도 화면 상단에 유지되도록 한다.
- 웨딩홀 추가 FAB가 하단 탭바와 겹치지 않도록 안전한 하단 오프셋을 적용한다.
- 하단 탭바 높이를 줄여 화면 점유율을 낮춘다.

## 원인
- `Dashboard`의 제목 영역이 일반 콘텐츠와 같은 스크롤 컨테이너 안에 있어 위로 사라진다.
- FAB는 고정 `bottom-24`만 사용하고 있어 실제 탭바 높이와 safe-area를 충분히 반영하지 못한다.
- `MobileLayout` 탭바는 세로 패딩이 커서 모바일 화면에서 차지하는 높이가 과하다.

## 구현 규칙
- `Dashboard` 제목 영역은 `sticky top-0` 구조로 유지되어야 한다.
- 제목 영역은 반투명 배경과 하단 여백을 가져서 스크롤 콘텐츠와 시각적으로 분리되어야 한다.
- `Dashboard` 본문 리스트는 제목 영역 아래에서 자연스럽게 스크롤되어야 한다.
- FAB는 탭바 높이와 `safe-area-inset-bottom`을 반영한 전용 하단 오프셋 유틸리티를 사용해야 한다.
- `MobileLayout` 탭바는 현재보다 더 얇은 세로 패딩을 사용해야 한다.
- 본문 하단 패딩(`pb-safe-nav`)도 줄어든 탭바 높이에 맞춰 함께 조정해야 한다.

## 검증 기준
- `Dashboard.tsx`에 sticky 제목 영역과 FAB 전용 하단 오프셋 클래스가 존재한다.
- `MobileLayout.tsx`의 탭바 세로 패딩이 기존보다 줄어든 구조를 가진다.
- `index.css`에 FAB 하단 오프셋과 조정된 nav padding 유틸리티가 존재한다.
- `npm test -- tests/unit/dashboard-layout-polish.test.ts`가 통과한다.
