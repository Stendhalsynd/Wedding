# S3-7 Android System UI And Safe-Top Headers

## 목표
- Android APK에서 status bar 영역의 시간, 배터리, 알림 아이콘이 읽히도록 한다.
- Android에서 status bar와 앱 헤더가 겹치지 않도록 시스템 UI와 safe-top 여백을 정리한다.
- 투어, 지도, 비교, 설정의 타이틀 영역을 상단에 고정하고 본문만 스크롤되게 한다.
- 뒤로가기 버튼이 있는 상세 헤더도 상단 시스템 영역과 겹치지 않도록 한다.

## 원인
- Android WebView가 edge-to-edge 상태로 렌더링되며 status bar 위로 콘텐츠가 올라간다.
- 상단 시스템 영역 배경과 앱 배경이 모두 밝아서 상태 아이콘 가독성이 매우 낮다.
- 일부 화면은 sticky header 구조가 없거나 safe-top padding이 없다.

## 구현 규칙
- Android native 환경에서는 Capacitor Status Bar 설정을 사용해 status bar를 앱 콘텐츠 위에 겹치지 않게 한다.
- Android status bar는 어두운 배경색과 밝은 아이콘 스타일을 사용한다.
- Android 15 targetSdk 환경을 고려해 edge-to-edge 강제 정책을 opt-out 한다.
- 투어, 지도, 비교, 설정 화면은 sticky header를 가져야 한다.
- 상세 화면의 뒤로가기 헤더는 safe-top padding을 포함해야 한다.
- safe-top 관련 spacing은 CSS 유틸리티로 관리한다.

## 검증 기준
- `package.json`에 `@capacitor/status-bar` 의존성이 존재한다.
- `src/services/systemUi.ts`가 Android native에서 status bar overlay, style, backgroundColor를 설정한다.
- `App.tsx`가 앱 시작 시 system UI 초기화를 호출한다.
- Android `styles.xml`에 status bar 색상/아이콘 및 `windowOptOutEdgeToEdgeEnforcement` 설정이 존재한다.
- `Dashboard.tsx`, `Map.tsx`, `Compare.tsx`, `Settings.tsx`, `HallDetail.tsx`가 safe-top sticky header 구조를 가진다.
- `npm test -- tests/unit/android-system-ui.test.ts tests/unit/sticky-screen-headers.test.ts`가 통과한다.
